## Available APIs

The l10n_mapper system provides **two ways** to access translations:

### 1. Direct Property Access (Standard Flutter)
```dart
// Access via getter - NOT a method call
final text = context.l10n.cashierDeposit;
final text2 = context.l10n.cashierWithdraw;

// Note: context.l10n('key') is NOT valid - l10n is a getter, not a method
```

### 2. Dynamic Key Lookup (l10n_mapper)
```dart
// Access via string key - useful for dynamic scenarios
final text = context.parseL10n('cashierDeposit');
final text2 = context.parseL10n('cashierWithdraw');

// With parameters
final text3 = context.parseL10n('cashierMinimumDeposit', arguments: [100, 'USD']);
```

## Workflow #1: Direct Property Access

### Code Example
```dart
final text = context.l10n.cashierDeposit;
```

### Step-by-Step Flow

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. context.l10n                                                 │
│    └─> BuildContextExtension.l10n getter                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 2. AppLocalizations.of(context)!                                │
│    └─> Calls Flutter's Localizations.of<AppLocalizations>()     │
│    └─> Looks up in widget tree                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 3. Returns AppLocalizations instance (e.g., AppLocalizationsEn) │
│    └─> Based on current locale (en, de, ar, etc.)               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 4. .cashierDeposit property access                              │
│    └─> Directly calls getter on AppLocalizationsEn              │
│    └─> Returns hardcoded string from class                      │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                    "Deposit" ✅
```

### Code Path
```dart
// In BuildContextExtension (generated by l10n_mapper)
extension BuildContextExtension on BuildContext {
  AppLocalizations get l10n => AppLocalizations.of(this)!;
}

// In AppLocalizations (generated by Flutter)
abstract class AppLocalizations {
  static AppLocalizations of(BuildContext context) {
    return Localizations.of<AppLocalizations>(context)!;
  }
  
  String get cashierDeposit; // Abstract getter
}

// In AppLocalizationsEn (generated by Flutter)
class AppLocalizationsEn extends AppLocalizations {
  @override
  String get cashierDeposit => 'Deposit';  // ← Direct return
}
```

### Performance
- ⚡ **Very Fast**: Direct property access
- 🎯 **No overhead**: No map lookups, no caching needed
- ✅ **Type-safe**: Compile-time checking

### When to Use
- ✅ When you know the key at compile time
- ✅ When you want type safety and IDE autocomplete
- ✅ Standard Flutter localization approach

## Workflow #2: Dynamic Key Lookup

### Code Example
```dart
final text = context.parseL10n('cashierDeposit');
```

### Step-by-Step Flow (WITH Performance Optimization)

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. context.parseL10n('cashierDeposit')                          │
│    └─> BuildContextExtension.parseL10n() method                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 2. Get AppLocalizations instance                                │
│    └─> AppLocalizations.of(context)!                            │
│    └─> Returns AppLocalizationsEn (for English locale)          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 3. Call localizations.parseL10n('cashierDeposit')              │
│    └─> AppLocalizationsExtension.parseL10n()                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 4. Call lookupKey('cashierDeposit')                             │
│    └─> Switch expression on key                                │
│    └─> O(1) compiler-optimized jump table                       │
│    └─> No map, no cache, zero allocation                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 5. Match key → direct property access                           │
│    └─> 'cashierDeposit' => localizations.cashierDeposit         │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ 6. Return value                                                  │
│    └─> String or null (key not found)                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                    "Deposit" ✅
```

### Code Path
```dart
// Step 1: Extension method
extension BuildContextExtension on BuildContext {
  String parseL10n(String translationKey, {List<Object>? arguments}) {
    final localizations = AppLocalizations.of(this)!;
    return localizations.parseL10n(translationKey, arguments: arguments);
  }
}

// Step 3-6: Switch-based lookup
extension AppLocalizationsExtension on AppLocalizations {
  Object? lookupKey(String key, [List<Object>? args]) {
    return switch (key) {
      'cashierDeposit' => cashierDeposit,
      'cashierMinimumDeposit' => switch (args) {
        [final Object amount, final Object currency] =>
          cashierMinimumDeposit(amount, currency),
        _ => throw ArgumentError('cashierMinimumDeposit requires 2 arguments'),
      },
      _ => null,
    };
  }

  String parseL10n(String translationKey, {List<Object>? arguments}) {
    final result = lookupKey(translationKey, arguments);
    if (result == null) return 'Translation key not found!';
    return result as String;
  }
}
```

### Performance

#### Every Call (consistent)
```
Time: ~2-3μs per lookup
Actions:
  1. Get localizations: ~1μs
  2. Switch on key: ~1μs (compiler jump table)
  3. Direct property/method access: ~1μs

No map allocation, no cache, no warm-up.
```

### When to Use
- ✅ When you need dynamic key lookup (key determined at runtime)
- ✅ When key comes from API, database, or user input
- ✅ When building translation management tools
- ✅ When you need to iterate through translations

## Performance Comparison

### Current: Switch-Based Lookup ✅

```dart
// Every call uses switch expression - zero allocation
context.parseL10n('cashierDeposit');    // Switch → direct access
context.parseL10n('cashierWithdraw');   // Switch → direct access
context.parseL10n('cashierBalance');    // Switch → direct access

// Total: 0 map entries, ~2-3μs per lookup
```

**Workflow**:
```
Call parseL10n
  ↓
lookupKey(key) → switch expression (O(1))
  ↓
Direct property/method access
  ↓
Return value
```

### Numbers

| Operation | Time |
|-----------|------|
| Per lookup | ~2-3μs |
| 100 lookups | ~200-300μs |
| 3000 lookups | ~4-12ms |

## Visual Flow Diagrams

### Multi-Locale Scenario

```
User opens app (English locale)
  ↓
context.parseL10n('deposit')
  ↓
lookupKey('deposit') → switch → localizations.cashierDeposit
  ↓
Return "Deposit"

───────────────────────────────────────

User switches to German
  ↓
context.parseL10n('deposit')
  ↓
lookupKey('deposit') → switch → localizations.cashierDeposit (de instance)
  ↓
Return "Einzahlung"

No cache, no map — each lookup is O(1) switch + direct access.
```

### Parameterized Translations

```dart
// Translation with parameters
context.parseL10n('cashierMinimumDeposit', arguments: [100, 'USD']);
```

**Flow**:
```
parseL10n('cashierMinimumDeposit', [100, 'USD'])
  ↓
lookupKey('cashierMinimumDeposit', [100, 'USD'])
  ↓
Switch on key → switch on args: [amount, currency]
  ↓
Direct call: localizations.cashierMinimumDeposit(100, 'USD')
  ↓
Returns: "The minimum deposit is 100 USD"
```

## Summary

### Quick Reference

| Feature | `context.l10n.key` | `context.parseL10n('key')` |
|---------|-------------------|---------------------------|
| **Type** | Property access | Method call |
| **Speed** | ⚡⚡⚡ Instant | ⚡⚡ Very fast (~2μs) |
| **Use Case** | Static keys | Dynamic keys |
| **Type Safety** | ✅ Yes | ❌ No (string key) |
| **Autocomplete** | ✅ Yes | ❌ No |
| **Runtime Key** | ❌ No | ✅ Yes |
| **Overhead** | None | ~2μs per lookup |

### When to Use Each

**Use `context.l10n.key`** when:
- You know the key at compile time
- You want IDE autocomplete
- You want type safety
- Maximum performance is critical

**Use `context.parseL10n('key')`** when:
- Key is determined at runtime
- Key comes from API/database
- Building translation tools
- Need to iterate through translations
- Key is constructed dynamically

## Common Misconceptions

### ❌ WRONG: `context.l10n('key')`
```dart
// This is NOT valid - l10n is a getter, not a method
final text = context.l10n('deposit'); // ERROR!
```

### ✅ CORRECT: Two separate APIs
```dart
// Option 1: Direct property access
final text = context.l10n.cashierDeposit;

// Option 2: Dynamic key lookup
final text = context.parseL10n('cashierDeposit');
```

---

**That's the complete workflow!** The switch-based approach provides fast dynamic key lookup (~2μs) with zero allocation.
